# Contributing to magic-cubes

## Open an issue first

Before you start working on a new feature, it is recommended that you open an issue to discuss whether your idea is a good fit for this package or whether I have already started working on it.

## Document your code

If you add new functionality, please document it accordingly and update the manual if necessary.

## Use of AI

`magic-cubes` has been developed without AI, and I would like to keep it that way.

You are, of course, welcome to use AI for guidance, just as you might ask for help on a forum.
However, I do not want to merge code that has been entirely designed and implemented by an AI agent.
Likewise, please write your pull request descriptions and issue reports yourself.

Using AI to proofread or correct your documentation is acceptable, but you must write the original content yourself.
